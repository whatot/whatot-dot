# Paper Overview

用于理解或评审单篇学术论文、预印本或研究报告。它可以做轻量相关工作检索来判断
新颖性和定位，但默认不是多篇论文综述。

## 核心原则

- 用户给出 URL、arXiv ID、PDF、HTML 页面或本地文件时，先读取来源再判断。
- 遇到付费墙或无法访问全文时，使用可访问内容，例如摘要、公开 PDF、预印本镜像或项目页，并明确说明限制。
- 区分作者声称、论文证据和自己的评价。
- 不只总结，要判断主要主张（claim）是否被方法和实验充分支持。
- 评价必须具体、可定位、可解释，优先引用章节、图、表、公式、实验或附录。
- 给出建设性建议：指出问题时尽量说明如何修复。
- 不按作者声誉、机构或发表状态评价论文，只评价工作本身。

## 使用场景

使用这个 skill 当用户：

- 提供论文 URL、arXiv 链接、DOI、会议/期刊页面、PDF 或本地论文文件。
- 要求分析论文、阅读论文、总结研究、评审论文、写 peer review。
- 想理解论文的贡献、方法、实验、优缺点、实践价值或可信度。

## 三阶段流程

### 阶段 1：理解论文

1. 识别论文元数据：
   - 标题、作者、年份、版本、会议/期刊/预印本状态。
   - 领域、子领域、论文类型。

2. 系统阅读：
   - 摘要和引言：动机、问题、贡献声称。
   - 相关工作：作者如何定位自己的工作。
   - 方法：核心机制、假设、输入输出、模型/系统/证明/数据流程。
   - 实验和结果：数据集、基线、指标、消融、定性结果。
   - 讨论、局限和附录：作者承认的边界、证明、实现细节。
   - 结论：结论是否超出了证据范围。

3. 提取核心主张：
   - 每个主张都要配对应证据。
   - 标记证据强度：强 / 中 / 弱 / 未支持。

### 阶段 2：批判分析

1. 轻量文献定位。
   - 当用户要求评审、新颖性判断或实践采用建议时，搜索论文标题、核心方法、基准（benchmark）、近期综述（survey）或 SOTA 比较。
   - 用于判断定位和比较是否公平，不要展开成完整文献综述。
   - 不要只做一次搜索；至少从论文标题、核心方法、基准或相关综述中选择 2-3 个角度交叉检查。
   - 关键外部来源要读取正文或足够完整的页面，不要只依赖搜索摘要。

2. 方法学评估。
   - 技术可靠性（Soundness）：技术是否成立，推理是否有漏洞。
   - 新颖性（Novelty）：真正的新东西是什么，是否只是增量改进。
   - 可复现性（Reproducibility）：实现细节、代码、数据、超参是否足够复现。
   - 实验设计（Experimental Design）：基线、数据集、指标、消融是否合理。
   - 统计严谨性（Statistical Rigor）：是否有多次运行、误差、显著性或不确定性分析。
   - 可扩展性（Scalability）：计算成本、部署成本、规模化证据是否充分。

3. 贡献等级判断：
   - Landmark：范式变化或广泛影响的突破。
   - Significant：扎实推进领域，有强证据支持。
   - Moderate：有用且有效，但边界明显或偏增量。
   - Marginal：提升很小、范围窄或和已有工作差异弱。
   - Below threshold：存在根本缺陷或证据不足。

4. 优缺点分析。
   - 是什么（What）：具体优势、问题、主张或风险。
   - 在哪里（Where）：章节、图、表、公式、实验或附录位置。
   - 为什么重要（Why）：说明它如何影响论文结论、可信度或实用价值。
   - 如何改进（How）：如果可修复，说明可操作的改进方向。

5. 来源质量和置信度。
   - 优先级：论文正文 / 附录 / 作者代码和数据 > 会议或期刊页面 > 官方项目页 > 近期 survey > 可靠技术博客 > 社区讨论。
   - 对重要判断标注置信度：高 / 中 / 低。
   - 高：来自论文或官方材料，并被实验、附录或多个独立来源支持。
   - 中：来自单一可靠来源，或论文证据充分但外部定位有限。
   - 低：依赖不完整文本、社区讨论、过时资料或推断。
   - 如果外部来源和论文说法冲突，明确列出冲突，不要用一个来源覆盖另一个来源。

### 阶段 3：综合输出

- 输出一份结构化论文分析，使用 [references/analysis-template.md](references/analysis-template.md) 的顺序。
- 根据论文内容和用户问题压缩空项，但不要跳过核心判断：主张、证据、方法学、局限和实践启发。
- 多篇论文比较：保持同一套评价标准，不把它扩展成综述，除非用户明确要求。

## 论文类型适配

| 类型               | 分析重点                                                 |
| ------------------ | -------------------------------------------------------- |
| 实证论文           | 实验设计、基线、指标、消融、统计显著性、可复现性。       |
| 理论论文           | 证明正确性、假设合理性、边界紧致性、边界条件、实践关联。 |
| 综述论文           | 覆盖面、分类体系、时效性、综合洞察、遗漏的重要方向。     |
| 系统论文           | 架构决策、真实 workload、可扩展性、部署证据、工程权衡。  |
| 数据集 / Benchmark | 数据收集、泄漏、标注质量、代表性、指标有效性。           |
| 方法 / 算法        | 问题定义、机制、复杂度、假设、比较公平性。               |
| 立场论文           | 论证一致性、证据质量、反方观点、影响范围。               |

如果类型不明确，基于论文的主要证据形态做推断，并说明这是推断。

## 输出要求

- 默认用 Markdown 输出一份完整但不过度冗长的论文分析。
- 使用 [references/analysis-template.md](references/analysis-template.md) 作为输出骨架。
- 用户只要快速了解时，可以压缩篇幅，但仍保留总体判断、方法、贡献、证据、局限和实践启发。
- 只有用户明确要求正式 peer review 时，才输出 Accept / Reject 这类审稿推荐。
- 不要沿用外部 skill 的 `/mnt/...` 保存或 `present_files` 约定，除非当前环境明确要求。

## 评审纪律

- 不要求不合理规模的额外实验，区分主要问题和小问题。
- 不因为论文没有解决另一个问题而扣分。
- 不只根据引言判断新颖性；新颖性重要时要查直接相关工作。
- 不评审“你希望存在的论文”，只评审当前论文。
- 不让写作质量压过技术贡献，除非表达问题妨碍判断工作本身。
- 对不能访问的部分保持明确边界，例如附录、证明、代码、数据或完整 PDF。
- 未发表或用户上传的非公开论文按保密材料处理。
- 发现数据偏见、双重用途（dual-use）、隐私或伦理风险时，建设性指出影响和缓解方向。
- 语气保持专业、克制、尊重；批评具体问题，不给作者贴标签。
- 把事实、作者观点、外部来源观点和自己的推断分开写；不要混成一个确定结论。

## 质量检查

评审式回答完成前检查：

- 主要主张已明确列出，并逐一对应证据强度。
- 已尽量完整阅读论文，而不是只读摘要和引言；未检查的部分要说明。
- 优势和弱点都具体、可定位，并解释影响。
- 评审模式下尽量提供至少 3 个优势和 3 个弱点；论文内容不足时说明原因。
- 方法学评分有简短理由，不是孤立分数。
- 有主要弱点，也有小问题，二者不要混在一起。
- 作者问题是真问题，不是伪装成问题的批评。
- 新颖性或推荐意见重要时，已经做轻量相关工作定位。
- 重要外部来源按质量排序使用，并记录来源冲突。
- 关键判断带有高 / 中 / 低置信度，低置信度判断说明原因。
- 改进建议具体、建设性、可执行。
- 语气专业尊重。
- 推荐意见、贡献等级和置信度与证据一致。
